Antrean Laporan Masuk

Klik header kolom untuk mengurutkan · Default: FIFO (terlama di atas)

Objek Layar Tindakan Deskripsi & Respons Sistem
Tabel Antrean Laporan Klik pada baris / tombol "Detail" Mengarahkan ke halaman detail laporan untuk validasi. Sistem menampilkan foto, lokasi GPS, dan deskripsi lengkap laporan.
Header Kolom (ID, Judul, Pelapor, dll) Klik pada header kolom Mengurutkan data berdasarkan kolom tersebut (ascending/descending). Default: FIFO berdasarkan Tanggal Masuk.
Input Pencarian Ketik ID, Nama Pelapor, atau Wilayah Sistem memfilter tabel secara real-time. Jika tidak ada hasil, sistem menampilkan pesan "Tidak ada laporan yang sesuai".
Dropdown Filter Status Pilih status (Diterima/Ditolak/Revisi/dll) Sistem memfilter tabel untuk menampilkan hanya laporan dengan status terpilih. Bisa dikombinasikan dengan pencarian teks.
Badge Prioritas (Kritis/Tinggi/Sedang) Read-only (tidak bisa diklik) Menampilkan tingkat prioritas laporan. Warna: Merah (Kritis), Oranye (Tinggi), Kuning (Sedang), Hijau (Rendah).
Badge Status Read-only (tidak bisa diklik) Menampilkan status validasi laporan. Hijau (Diterima), Kuning (Dalam Peninjauan/Revisi), Merah (Ditolak).
Tombol "Detail" (Aksi) Klik / Tap Mengarahkan ke halaman /admin/dashboard/detail?id=[ID] untuk melihat detail lengkap dan melakukan validasi (Terima/Tolak/Revisi).
Teks SLA (Sisa Waktu) Read-only Menampilkan sisa waktu SLA (Service Level Agreement). Jika "Sisa 6 jam" = warna oranye, "-" (melebihi batas) = warna merah.

Dokumentasi UI Merge Tiket

Penjelasan komponen dan aksi pada fitur penggabungan tiket duplikat

Objek Layar Tindakan Deskripsi & Respons Sistem
Tabel Daftar Tiket Duplikat Klik checkbox pada tiket Memilih tiket yang akan digabungkan. Sistem menampilkan foto, similarity score (%), dan status tiket.
Input Pencarian (Merge) Ketik ID, Judul, atau Lokasi Sistem memfilter daftar tiket secara real-time. Tiket yang tidak cocok akan disembunyikan.
Badge Similarity Score Read-only Menampilkan persentase kemiripan (68%, 72%, dll). Warna: Merah (>85%), Oranye (70-85%), Kuning (60-70%), Hijau (<60%).
Status Badge (Disposisi/Dikerjakan) Read-only Ungu = Disposisi (tidak bisa digabung), Kuning = Sedang Dikerjakan (tidak bisa digabung), Abu-abu = Menunggu (bisa digabung).
Tombol "Review Terpilih" Klik setelah memilih ≥2 tiket Membuka modal review untuk memilih Parent Ticket (tiket utama) dan mengisi alasan penggabungan.
Modal Review & Merge Pilih Parent Ticket & isi alasan Parent Ticket adalah tiket yang akan menjadi utama. Child tickets akan disembunyikan dari antrean. Alasan minimal 10 karakter.
Cluster Tiket (Kanan) Klik "Pisahkan" pada cluster Menampilkan tiket yang sudah digabung dalam kelompok. Tombol "Pisahkan" mengembalikan tiket ke daftar utama.
Foto Bukti Tiket Read-only (tidak bisa diklik) Menampilkan foto laporan dari warga sebagai referensi kemiripan visual antar tiket.
Alert Messages Read-only (auto-hide setelah refresh) Hijau = Berhasil digabung, Biru = Berhasil dipisahkan, Merah = Error (tiket sudah disposisi/dikerjakan, belum pilih parent, dll).
